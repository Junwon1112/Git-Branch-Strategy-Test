-문서 소개-
해당 문서는 공부했던 깃 전략들을 텍스트 문서로 실적용해보고 연습해보기 위해 만들어 졌습니다. 
"A successful Git branching model" 이란 컬럼에 소개된 전략을 바탕으로 작성되었습니다.

원문 링크 : http://nvie.com/posts/a-successful-git-branching-model/

-개요-
3개 항목으로 이루어진 자기소개서를 작성할 것이며 5가지 주요 브랜치로 만들어 협업 상황을 가정하고 깃을 사용할 것 입니다.

-주요 브랜치 항목 및 사용 이유-
1. Main 브랜치 - 사용할 수 있는 기능 단위로 구현된 공개될 브랜치, 각 브랜치에는 '버전 태그'가 붙을 것
2. Develop 브랜치 - 개발 단위에서 주로 사용할 브랜치, Topic 브랜치와(기능 구현 하는 브랜치) release 브랜치 에서 기본적으로 merge되는 브랜치
3. Topic 브랜치 - 개인별 실제 기능을 구현하는 브랜치, 여기서는 자소서 항목들의 내용을 써나갈 예정
4. release 브랜치 - 버그 수정을 해서 Main과 Develop브랜치에 merge되는 브랜치
5. hotfix 브랜치 - 실제 서비스 중 심각한 버그가 발생시 Main브랜치에 바로 수정되어 들어가는 브랜치
